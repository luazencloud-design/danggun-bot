# 🥕 당근 자동응답 봇 — 사용 설명서

당근마켓 판매 채팅을 Gemini AI가 대신 응대해 주는 프로그램이에요. 이 설명서 하나만 보고도 설치부터 실사용까지 전부 따라올 수 있게 썼습니다.

---

## 1. 🖥️ 처음 설치하기 (다른 PC로 옮겼을 때)

### 1-1. Node.js 설치 (단 한 번만)
이 프로그램은 Node.js가 있어야 실행됩니다. 컴퓨터에 없으면 먼저 설치하세요.

**방법 A: 자동 설치 (Windows 10/11, 명령어 한 줄)** ⭐ 추천
1. `Win + X` → **"터미널(관리자)"** 또는 **"PowerShell(관리자)"** 클릭
2. 아래 명령 복사 붙여넣고 Enter:
   ```
   winget install OpenJS.NodeJS.LTS
   ```
3. 약관 `Y` 입력 → 설치 자동 완료 (1~2분)
4. 터미널 닫고 새 cmd 열어서 `node --version` 확인

**방법 B: 수동 설치 (winget이 안 되는 경우)**
1. https://nodejs.org/ko 접속
2. 왼쪽 초록색 **"LTS" 버튼 클릭** (권장 버전, 20 이상 아무거나 OK)
3. 다운로드된 `.msi` 파일 실행 → "Next" 연타 → 설치 완료
4. 설치 확인: `Win + R` → `cmd` → `node --version` → 버전 숫자가 나오면 OK

### 1-2. 이 폴더 복사
`danggeun-autobot` 폴더 전체를 원하는 위치에 복사하세요. **경로에 한글이나 공백이 있어도 OK**지만, 짧은 영문 경로 권장 (예: `C:\autobot`).

### 1-3. 첫 실행
폴더 안의 **`실행.bat`** 파일을 더블클릭하세요.

- 최초 1회만 의존성 자동 설치 (1~2분 걸림, 인터넷 필요)
- 완료되면 검정 창에 `Local: http://localhost:5173` 줄이 뜨고 브라우저가 자동으로 열려요
- 검정 창은 **닫지 말고 유지** — 닫으면 봇도 꺼집니다

### 1-4. Gemini API 키 입력
처음 열면 팝업이 뜹니다.

1. https://aistudio.google.com/app/apikey 접속 (구글 로그인)
2. **"Create API key"** 클릭 → 생긴 키 복사 (`AIza...`로 시작)
3. 팝업 입력칸에 붙여넣기 → **"저장하고 시작"** 클릭

> **무료 할당량**: Gemini 2.5 Flash 기준 하루 1,500 요청까지 무료. 일반 사용엔 충분.
> **API 키 보안**: 키는 본인 브라우저에만 저장되며 외부로 전송되지 않아요.

## 2. 📦 상품 등록하기

**상단 "설정" 탭 → 왼쪽 "상품 관리" 패널**

### 2-1. 상품 추가
- `+ 상품 추가` 버튼으로 새 카드 생성
- 여러 상품을 동시에 관리 가능 (업로드한 게시글 수만큼)

### 2-2. 각 상품 카드에서 입력
| 항목 | 설명 |
|---|---|
| **상품 라벨** | 내가 알아보기 쉬운 이름 (예: "아이폰", "그래픽카드") |
| **봇 ON/OFF** | 이 상품 전체를 봇이 응대할지 여부 |
| **글 제목** | ⭐ 가장 중요 — **당근 채팅방 상단에 뜨는 게시글 제목 그대로 복붙** |
| 상품명 | 봇이 대화에서 쓸 상품 호칭 |
| 판매가 / 마지노선 | 마지노선 미만으로는 절대 할인 안 됨 |
| 네고 정책 | 불가 / 소폭 / 협의 중 선택 |
| 택배 · 직거래 | 각 토글 + 안내 문구 |
| 말투 | 친근 / 정중 / 쿨 |
| 추가 규칙 | 자유 입력 (예: "교환·환불 불가") |

### 2-3. URL 자동 불러오기 (있으면 편리)
상품 카드 위쪽 `URL` 탭에 당근 게시글 URL 붙여넣으면 AI가 제목·가격·상태를 자동 추출해서 채워줍니다. 접근 차단되면 `붙여넣기` 탭에서 본문 텍스트 복붙.

### 2-4. 저장
설정 변경 후 하단 **"전체 저장"** 눌러야 반영. 브라우저 로컬 저장이라 닫아도 유지돼요.

## 3. 🎯 글 제목 매칭이 동작하는 방식

봇이 **"이 채팅방은 어느 상품 얘기지?"** 를 판별할 때 이 매칭을 씁니다.

### 규칙
- **공백 전부 무시**: `"아이폰 14Pro"` ≈ `"아이폰14Pro"`
- **부분 문자열 허용**: 앞뒤로 다른 글자가 붙어도 OK
- **오타는 거부**: 한 글자라도 다르면 매칭 안 됨

### 예시 (설정한 글 제목: `아이폰 14 Pro 256GB 딥퍼플`)
| 실제 채팅방 제목 | 매칭? | 왜 |
|---|---|---|
| `판매완료 아이폰 14 Pro 256GB 딥퍼플` | ✅ | 앞에 "판매완료" 붙어도 본문 포함됨 |
| `아이폰 14 Pro 256GB 딥퍼플`\n`디지털기기 · 10개월 전` | ✅ | 뒤에 태그·시간 붙어도 OK |
| `아이폰 14 Pro 256GB 딥퍼를` | ❌ | "플" → "를" 오타 (의도적 방지) |

## 4. 💬 채팅방 관리 패널 (설정 탭 오른쪽)

여기서 **어느 고객의 어느 채팅방에 봇을 켤지**를 제어합니다.

### 수동 관리 (현재)
- `+ 추가` 버튼으로 채팅방 슬롯 생성
- 닉네임 + 글 제목 입력
- **매칭된 상품** 드롭다운:
  - `자동 매칭` (기본) — 글 제목 기반으로 자동 선택
  - 특정 상품 이름 — 수동 강제 지정 (자동 매칭 오류 방지)
- **이 채팅방 봇 자동 응답** 토글 — 고객별 ON/OFF

### 자동 관리 (크롬 확장 설치 시, 예정)
- 당근 채팅 사이드바의 모든 대화가 여기에 자동 등록
- 새 메시지 오면 봇이 자동 응답
- 상태 실시간 업데이트 (자동 / 수동 / OFF)

## 5. 🧪 채팅 테스트 (상단 "채팅 테스트" 탭)

실제 당근 연동 전에 봇 응답을 미리 볼 수 있는 시뮬레이터.

1. 상단 파란 박스의 **"채팅방 글 제목"** 입력칸에 테스트할 게시글 제목 넣기
2. 왼쪽 사이드바에 **매칭된 상품**이 표시되는지 확인
3. 아래 **빠른 질문 버튼** 또는 직접 입력으로 구매자 시점 메시지 전송
4. 봇 응답 확인

### 수동 모드(핸드오프)로 전환되는 경우
다음을 감지하면 봇이 즉시 멈추고 상단 빨강 배너를 띄웁니다:

- **키워드 감지 (API 호출 전 즉시)**: 계좌이체, 카톡, 전화번호, 외부결제, 해외배송, 이메일/링크, 페이팔·위챗 등
- **AI 판단 (Gemini 응답 해석)**: 실물 사진 요청, 개인정보 질문, 구매 확정, 모르는 스펙 질문, 분쟁·환불 얘기

이때 **입력창은 비어있는 상태로 유지**돼서 실수로 경고문이 구매자에게 전송되는 일은 없어요. 직접 답하거나 상단 "봇 재개" 버튼을 누르세요.

## 6. 🔧 자주 묻는 문제

### ⚡ 봇이 답장을 안 해요 — 5분 진단 가이드

**Step 1. 확장 팝업에서 상태 확인** (가장 빠름)
1. 크롬 우측 상단 🥕 아이콘 클릭
2. **🩺 시스템 상태** 4개 타일 확인:
   - `Gemini API 키` → ✗ 미설정이면? 웹앱에서 API 키 입력 후 "전체 저장" 눌러주세요
   - `상품 등록` → ✗ 0개면? 웹앱 설정 탭에서 "상품 추가"하고 글 제목 입력
   - `동기화된 방` → 0이면? `chat.daangn.com` 탭을 열어두세요
   - `핸드오프 중` → N개면? 해당 방이 정지 상태. 방 열어서 "봇 재개" 누르세요
3. **📋 최근 결정** 섹션 확인:
   - `✅ 답변` — 정상 동작 (그런데 상대방이 받지 못했다면 → 전송 버튼 이슈)
   - `🚨 핸드오프` — 위험 감지로 정지 (이유 표시됨)
   - `⏸️ 건너뜀` — **왜 스킵됐는지 이유가 찍힘** (상품 미설정, 매칭 실패 등)
   - `⚠️ 오류` — Gemini 호출 실패 (키 잘못됨, 할당량 초과 등)
   - **기록이 아예 없음** → 메시지 감지 자체가 안 되고 있음 → Step 2로

**Step 2. chat.daangn.com 에서 F12 콘솔 진단**
1. 당근 채팅 열고 F12 → Console 탭
2. 아래 한 줄 붙여넣고 Enter:
   ```js
   __daangnBot.diag().then(console.log)
   ```
3. 결과 해석:
   ```
   page.room            ← /room/XXX 이어야 함 (방을 하나 열어야 함)
   page.title           ← 글 제목이 보여야 함 (null이면 글 제목 추출 실패)
   textareaExists       ← true (false면 입력창 selector 바뀜, 확장 업데이트 필요)
   listRegionExists     ← true (false면 메시지 리스트 selector 바뀜)
   storage.apiKey       ← "AIza…" (없음이면 웹앱에서 저장 안 됨)
   storage.products     ← 1 이상 (0이면 상품 미등록)
   ```

**Step 3. 수동으로 가짜 메시지 보내 chain 테스트**
```js
__daangnBot.test('상태 어떤가요?')
```
- 현재 방의 글 제목으로 상품 매칭 시도 → Gemini 호출 → 답변 주입까지 전 과정 실행
- 결과가 `{ ok: true }`면 chain 자체는 작동. 실제 메시지 감지만 문제
- 결과가 `undefined`면 background.js가 죽었을 수 있음 → 확장 새로고침

**Step 4. 확장 새로고침**
배경 스크립트나 매니페스트가 바뀌었는데 로드된 확장이 옛 코드면 아무리 고쳐도 안 돼요.
1. `chrome://extensions` 접속
2. 🥕 당근 자동응답 봇 카드의 **🔄 새로고침** 아이콘 클릭
3. chat.daangn.com 탭도 `Ctrl+Shift+R` 로 강제 새로고침
4. 웹앱 탭(localhost)도 새로고침

**Step 5. 실제 당근 채팅으로 재시도**
- 다른 계정에서 메시지 보내기 (같은 PC의 다른 브라우저 or 다른 디바이스)
- 🥕 팝업 열어서 "📋 최근 결정"에 새 항목이 뜨는지 확인
- 안 뜨면 메시지 감지 실패 (selector 문제일 가능성). 이 경우 F12 Console에 `[당근봇] 🔔 새 메시지:` 로그가 있는지 확인

### 🧩 흔한 원인 → 해결

| 증상 | 원인 | 해결 |
|---|---|---|
| 팝업에 "API 키 ✗ 미설정" | 웹앱에서 저장 안 됨 | 웹앱 키 입력 → "전체 저장" 누르기 |
| "상품 등록 ✗ 0개" | 상품 추가 안 함 | 웹앱 설정 탭에서 상품 추가 + 글 제목 필드 채우기 |
| 매번 "상품 매칭 실패" | 글 제목 오타/불일치 | 실제 당근 채팅방 상단 게시글 제목 그대로 복사해서 상품의 "글 제목" 필드에 붙여넣기 |
| "이 방 봇 OFF" | 방별 토글이 꺼짐 | 웹앱 "채팅방 관리"에서 해당 방 토글 ON |
| "핸드오프 중: ..." | 전에 위험 감지로 정지됨 | 방 열고 빨강 배너의 "봇 재개" 클릭 |
| Gemini 오류 "403" / "API_KEY_INVALID" | 키가 잘못되거나 무효 | aistudio.google.com에서 키 재발급 |
| Gemini 오류 "429" | 무료 할당량 초과 | 내일 기다리거나 유료 전환 |
| 답장은 하는데 전송 안 됨 | 전송 버튼이 비활성 상태 | F12 콘솔에 `전송 버튼 비활성화` 로그 있으면 buyer 메시지가 특수한 경우. 수동 답장 필요 |
| `Uncaught Error: Extension context invalidated` | 확장이 업데이트/재로드 됐는데 페이지 오래됨 | **페이지 상단 주황 배너의 "새로고침" 버튼** 클릭 (또는 Ctrl+R) — 크롬 확장을 수정/새로고침 하면 거의 항상 나오는 에러. 페이지 리로드 한 번이면 해결 |
| 상품 등록/API 키가 저장 안 됨 | Extension context invalidated 탓에 동기화 실패 | 위 에러와 같은 원인. **크롬 확장 새로고침 + 웹앱 페이지 새로고침** 한 번 하면 해결 |
| 배포용_만들기.bat에서 "IP는 내부 또는 외부 명령..." | Windows 11에서 WMIC 제거됨 (구 버전 bat) | 최신 버전은 PowerShell 기반으로 재작성됨. 배포 ZIP 다시 만들어서 써주세요 |

### 💡 테스트 팁
- **같은 PC + 2개 계정**: 크롬에 메인 계정 로그인 (확장 + 봇용), 다른 브라우저(엣지/파이어폭스/시크릿 모드)에 구매자 계정 로그인, 상품 클릭 → 채팅 시작 → 메시지 보내기
- **확장이 감지하는 건 "메인 계정 브라우저에 들어오는 메시지"**. 구매자 계정에서 보내는 메시지는 본인 브라우저에선 `.for-containment.right` 이라 무시됨 (정상 동작)

## 6-1. 💬 기타 자주 묻는 문제

**Q. 검정 창이 "Node.js is not installed"라고 나와요**
→ 1-1 단계 다시 보고 Node.js 설치. 설치 후 재부팅 권장.

**Q. "연결이 불안정했네요" 또는 "❌ API 키 문제"가 뜨면?**
→ F12 눌러 개발자 도구 → Console 탭에 빨강 글씨 확인. 대부분은:
 - Gemini API 키 잘못 입력 → `Gemini 키` 버튼 눌러 재입력
 - 하루 무료 할당량 초과 → 내일 기다리거나 Google Cloud 결제 설정
 - 인터넷 연결 불안 → 와이파이 확인

**Q. 브라우저가 자동으로 안 열려요**
→ 검정 창에 뜬 `Local: http://localhost:5173` 주소를 직접 열어보세요.

**Q. 포트 5173이 이미 사용 중이래요**
→ 다른 프로그램(혹은 이전 실행창)이 쓰는 중. 작업관리자에서 `node.exe` 모두 종료 후 다시 실행.

**Q. 설정이 저장 안 돼요**
→ 브라우저 시크릿 모드에서 열면 저장 안 됨. 일반 모드로 다시 열기. 매번 "전체 저장" 버튼 클릭 필요.

**Q. 다른 PC로 옮기려면 뭘 가져가야 하나요?**
→ `danggeun-autobot` 폴더 전체(단, `node_modules`는 용량 크니 제외 OK). 새 PC에서 `실행.bat` 돌리면 자동으로 재설치됩니다. **설정(상품·고객 정보)은 브라우저별로 저장**되니 새 PC에서는 다시 입력해야 합니다.

**Q. 설정을 백업하고 싶어요**
→ F12 → Console에 `copy(localStorage.getItem('danggeun_autobot_config_v2'))` 입력 엔터 → 텍스트 파일에 저장. 복원은 `localStorage.setItem('danggeun_autobot_config_v2', '여기에_붙여넣기')`.

## 8. 🧩 크롬 확장 프로그램 설치 (실제 당근 연동)

웹앱(설정·테스트 화면)은 `실행.bat`으로 동작하지만, **실제 당근 채팅을 자동으로 읽고 답장하려면 크롬 확장을 1회 설치**해야 합니다.

### 8-1. 설치 (1분)

1. **`크롬확장_설치.bat` 더블클릭**
   - 자동으로 크롬 `chrome://extensions` 페이지가 열립니다
   - 확장 폴더 경로가 클립보드에 복사됩니다
2. 우측 상단 **"개발자 모드"** 토글을 **켭니다**
3. 좌측 상단 **"압축해제된 확장 프로그램을 로드합니다"** 버튼 클릭
4. 팝업 창 주소란에 **`Ctrl+V`** (클립보드에 복사된 `extension` 폴더 경로 붙여넣기) → **선택**
5. 확장 목록에 **🥕 당근 자동응답 봇**이 나타나면 성공

### 8-2. 확인
- `https://chat.daangn.com` 접속
- 크롬 우측 상단 **🥕 아이콘** 클릭 → 팝업에 채팅방 목록이 실시간으로 뜨면 정상
- F12 → Console에 `[당근봇] 사이드바 동기화 N개 방` 로그 확인

### 8-3. 현재 버전 (v0.1) 기능
- ✅ 사이드바 채팅방 목록 자동 추출
- ✅ 공식 계정 자동 구분 (배지 표시)
- ✅ 방 전환 실시간 감지
- ✅ 팝업에서 원하는 방으로 직접 이동
- 🚧 **다음 버전 예정**: 자동 응답, 핸드오프 배너, 웹앱-확장 설정 연동

### 8-4. 주의
- "개발자 모드"를 켜면 크롬 시작시 상단에 알림 바가 뜰 수 있어요. 기능엔 영향 없고, 해당 바의 "취소"로 끌 수 있습니다.
- 확장은 **본인 PC에서만 동작**하고, API 키와 설정은 외부로 전송되지 않습니다.
